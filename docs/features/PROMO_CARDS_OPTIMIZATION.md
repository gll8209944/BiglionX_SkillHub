# Widget 和 SDK 宣传优化方案

## 📋 优化概述

将原来占据页面顶部的大型横幅改为**简洁的双栏推广卡片**，放置在Skills页面的侧边栏筛选面板下方，使页面更加清爽、专业。

---

## 🎯 优化目标

1. **减少视觉干扰** - 移除顶部大型横幅，让页面更简洁
2. **提升用户体验** - 推广内容不打断用户浏览流程
3. **保持曝光度** - 在侧边栏固定位置展示，持续可见
4. **增强点击率** - 使用渐变色彩和悬停动画吸引注意

---

## 🔄 改动对比

### Before（之前）
```
┌─────────────────────────────────────┐
│   InteractiveWidgetDemo (大横幅)     │  ← 占据整个宽度，非常显眼
├─────────────────────────────────────┤
│   SDKPromoBanner (横幅)              │  ← 又一个横幅
├─────────────────────────────────────┤
│   导航栏                              │
├─────────────────────────────────────┤
│   搜索框                              │
├──────────┬──────────────────────────┤
│ 侧边栏    │   Skills 列表            │
│ 筛选器    │                          │
└──────────┴──────────────────────────┘
```

**问题：**
- ❌ 两个大横幅占据过多空间
- ❌ 页面显得杂乱
- ❌ 用户需要先关闭/跳过才能看到主要内容
- ❌ 移动端体验不佳

---

### After（现在）
```
┌─────────────────────────────────────┐
│   导航栏                              │  ← 干净简洁
├─────────────────────────────────────┤
│   搜索框                              │
├──────────┬──────────────────────────┤
│ 侧边栏    │   Skills 列表            │
│ 筛选器    │                          │
│          │                          │
│ ┌──────┐ │                          │
│ │Widget│ │                          │  ← 推广卡片
│ └──────┘ │                          │
```

**优势：**
- ✅ 页面顶部清爽，无干扰
- ✅ 推广内容自然融入布局
- ✅ 用户在筛选时会看到推广
- ✅ 响应式设计友好

## 📦 新增组件

### PromoCards.tsx

**位置：** `apps/web/components/ui/PromoCards.tsx`

**功能：**
- 双栏卡片布局（桌面端并排，移动端堆叠）
- 渐变背景 + 装饰性光效
- 悬停缩放动画（scale 1.02）
- 特性标签展示
- 直接链接到演示/文档页面

**设计特点：**
```tsx
// Widget 卡片 - 紫色渐变
bg-gradient-to-br from-indigo-500 via-purple-500 to-pink-500

// SDK 卡片 - 蓝色渐变  
bg-gradient-to-br from-blue-500 via-cyan-500 to-teal-500

// 悬停效果
hover:shadow-xl hover:scale-[1.02] transition-all duration-300

// 箭头动画
opacity-0 group-hover:opacity-100 group-hover:translate-x-1
```

## 📍 集成位置

### Skills 页面

**文件：** `apps/web/app/skills/page.tsx`

**修改内容：**

```tsx
  </div>

```

## 🔧 技术细节

### 响应式设计
```tsx
// 桌面端：双栏并排
grid md:grid-cols-2 gap-6

// 移动端：自动堆叠
// （md以下单列布局）
```

### 性能优化
- 使用CSS渐变而非图片背景
- 纯CSS动画（无JavaScript开销）
- 组件懒加载（Next.js默认）

### 可访问性
- 完整的Link标签（键盘可访问）
- 适当的aria标签
- 足够的颜色对比度
