# 邮箱登录功能测试指南

## ✅ 已实现的功能

### 1. 邮箱注册
- **路径**: `/register`
- **功能**: 
  - 邮箱格式验证
  - 密码强度验证（至少8位，包含字母和数字）
  - 密码 bcrypt 加密存储
  - 自动登录或跳转到登录页

### 2. 邮箱登录
- **路径**: `/login`
- **功能**:
  - 选项卡切换（GitHub / 邮箱）
  - 邮箱和密码验证
  - NextAuth Credentials Provider 集成
  - 登录成功后跳转到 Dashboard
  - 错误提示

## 🧪 手动测试步骤

### 测试 1: 新用户注册

1. 打开浏览器访问: `http://localhost:3001/register`
2. 选择"邮箱注册"标签
3. 填写表单:
   - 姓名: 测试用户（可选）
   - 邮箱: test@example.com
   - 密码: Test1234
   - 确认密码: Test1234
4. 点击"创建账户"
5. **预期结果**: 
   - ✅ 注册成功

## 🔐 安全特性

- ✅ 密码使用 bcrypt 加密（12轮哈希）
- ✅ 服务端密码验证
- ✅ NextAuth JWT session 管理
- ✅ Prisma ORM 防止 SQL 注入
- ✅ 邮箱格式验证
- ✅ 密码强度要求

## 📝 API 端点

### POST /api/auth/register
**请求体**:
```json
{
  "email": "user@example.com",
  "password": "SecurePass123",
  "name": "用户名"
}
```

**成功响应** (201):
```json
{
  "message": "注册成功",
  "user": {
    "id": "user-id",
    "email": "user@example.com",
    "name": "用户名"
  }
}
```

**错误响应** (400/409):
```json
{
  "error": "错误信息"
}
```

## 🐛 常见问题

### Q: 登录后没有跳转到 Dashboard？
A: 检查浏览器的开发者工具控制台是否有错误，确认 NextAuth 配置正确。

### Q: 注册时显示"该邮箱已被注册"？
A: 这是正常行为，表示该邮箱已经存在于数据库中。请直接登录。

### Q: 登录时显示"邮箱或密码错误"？
A: 请确认：
- 邮箱是否正确
- 密码是否正确（注意大小写）
- 用户是否已通过邮箱注册（不是 GitHub 注册）

### Q: 服务器启动失败？
A: 检查：
- 端口 3000/3001 是否被占用
- `.env.local` 文件中的环境变量是否正确配置
- 数据库连接是否正常
